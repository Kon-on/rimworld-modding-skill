# 工作流：批量处理 Mod 需求

> 适用：用户提供需求文件（requirements.md），批量添加多个武器/建筑/服装/物品等

## 流程概览

```
用户提供 requirements.md
  → ① 读取文件，解析所有需求
  → ② 处理第一个需求（加载对应模板生成）
  → ③ 展示结果给用户确认 ✅ / 修改 ✏️ / 跳过 ⏭️
  → ④ 用户确认后，继续下一个
  → ⑤ 全部完成后，输出汇总清单
```

## 第一步：解析需求文件

读取 `requirements.md`，按 `## 需求 N: <名称>` 分割每个需求，提取以下字段：

| 字段 | 必填 | 说明 |
|------|:---:|------|
| 类型 | ✅ | 近战武器 / 远程武器 / 建筑 / 服装 / 原材料 / 配方 |
| defName | ✅ | 全局唯一标识符（含前缀，如 `RCS_PlasmaSword`） |
| 标签 | | 用逗号分隔，如 `武器, 近战, 能量` |
| 材料 | | 如 `钢铁x50, 零部件x3` |
| 其他 | | 按类型对应的特有字段（见下方对照表） |

### 类型 ↔ 字段对照

| 类型 | 特有字段 |
|------|---------|
| 近战武器 | 伤害, 冷却, 科技等级, 护甲穿透 |
| 远程武器 | 伤害, 射程, 射速, 弹匣容量, 科技等级 |
| 建筑 | 工作量, 科技等级, 尺寸 |
| 服装 | 护甲(锐利/钝器/热能), 隔热, 科技等级 |
| 原材料 | 价值, 美观度, 市场价 |
| 配方 | 制作时间, 所需技能, 产量 |

### 需求文件格式参考

详见 `templates/requirements-template.md`。

## 第二步：逐个处理

每处理一个需求：

1. 根据 `类型` 找到对应模板
2. 加载模板，替换占位符
3. 生成完整的 Def / 代码
4. 展示给用户确认

### 确认选项

| 用户操作 | 含义 |
|---------|------|
| ✅ **确认** | 当前需求通过，进入下一个 |
| ✏️ **修改** | 用户提出修改意见，调整后重新展示 |
| ⏭️ **跳过** | 跳过当前需求，进入下一个 |

## 第三步：汇总输出

全部需求处理完成后，输出汇总清单：

```
=== 批量处理完成 ===

✅ 已确认:
  1. RCS_PlasmaSword - 等离子剑（近战武器）
  2. RCS_PrecisionBench - 精密组装台（建筑）

⏭️ 已跳过:
  3. RCS_StealthSuit - 隐身服（服装）- 用户跳过

=== 下一步 ===
将已确认的 Def 整合到 mod 项目目录中，进游戏测试。
```

## 注意

- 每个需求独立生成，互不干扰
- 用户确认前不写入 mod 目录
- 全部确认完成后统一告知如何整合
